CHAT APPLICATION

  • Category: Program
  • Programming Language: Python
  • Topic: Socket Programming, GUI

Detailed Problem statement

The objective of our program is to make a chat application with GUI using socket programming.
There are two programs for this chat application to work.
The first program is the program to be run on the server side. This program creates a host / server. This program should be run only on one device at a time.
The second program is the program to be run on the client side. This is the program through which we would be able to chat.This program can be run on multiple devices. When we run the program, it connects the device to the host. Thus you can send the message to the server. Later, the server sends the message received by the client to all the other clients connected to the server.
Thus real time communication takes place

Python concepts used by our project

  • Socket Programming
  • GUI using Tkinter
  • Multi-Threading
  • Exception Handling

Functionalities / features supported by project

  • You can type your message in the text box appearing in the window.
  • The “Send” button sends the message to the server.
  • The “Exit” button exits the program.
  • The Enter key is bonded with the Send button i.e. pressing the enter key means clicking the send button.
  • The escape key is bonded with the Exit button.
  • Others message is displayed on your window preceding by their IP address.
  • Your message is displayed on you window preceding by <YOU>.
  • New messages are displayed above the previous messages.

Working of the program

  1. Server side:
    • When the server program is run, it creates a host which has a unique IP address and a port number. The client can only connect to the server if he has the IP address and port number of the server.
    • The server code continuously keeps on searching for new clients.
    • As soon as a client is found, the server adds the client to the list of client and makes a new thread for the client. This thread continuously searches for a message sent from the client.
    • As soon as a message is received from the client, the broadcast function is called.
    • This function send the message received from the client to all the other clients in the list of clients.
  2. Client side:
    • The client connects to a server only if it has the IP address and port number of the server.
    • As the client is connected a window is opened and a new thread is formed.
    • This thread continuously keeps on checking if any message is sent by the server.
    • If a message is found then the program prints the message on the window.
    • You can send the messaged you have typed in the textbox using the send button.
    • This button sends the message to the server.
